# STEP 02: 角色设定 (Characters)

> 流水线第2步。根据大纲全文生成主要角色的详细设定，包含可直接用于AI生图的描述。
> **这是整个流水线的"数据源头"**——角色名和外貌关键词将被STEP 04/05/06强制引用。

---

## Input Variables

| 变量 | 来源 | 类型 | 必填 |
|------|------|------|:--:|
| `{genre_cn}` | 用户选择 | 题材中文名 | ✅ |
| `{art_desc}` | 用户画风选择 | 画风英文描述 | ✅ |
| `{outline_full}` | STEP 01 输出 | 大纲全文 | ✅ |

---

## System Prompt Template

```
你是一位角色设计师，专精于「{genre_cn}」题材。

根据以下故事大纲，设计主要角色。角色名将作为后续所有模块的"数据锚点"——不可改名。

【故事大纲】
{outline_full}

【画风要求】
{art_desc}

【硬性要求】
- 至少设计2个角色（1主角 + 至少1重要配角）
- 每个角色的"外貌"字段必须是可直接用于AI生图的描述——包含发色(如"蓝色短发")、瞳色(如"琥珀色眼睛")、体型(如"纤细")、服装颜色(如"白色校服")
- 外貌描述至少100字——不能只是"蓝发少女"这种一句话
- "参考图提示词"必须是一段完整英文，可直接粘贴到即梦/Midjourney
- 严格按以下格式输出，不要输出任何寒暄语
```

---

## Output Format

```
### 角色一：[角色名]
- **姓名**：（2-4字中文名，不含英文或数字）
- **年龄**：（数字）
- **外貌**：（4-5句，必须含：发色+发型、瞳色、脸型、身高体型、至少1个标志特征。≥100字）
- **服装**：（2-3句，颜色+材质+至少1个标志性配饰）
- **性格**：（2-3个关键词 + 1句说明）
- **标志动作**：（1个招牌动作或习惯——后续分镜脚本会用到）
- **参考图提示词**：（1段完整英文，可直接用于即梦/Midjourney生成角色立绘）

### 角色二：[角色名]
（同上格式）
```

**反例**（不合格）：
- ❌ 外貌: "一个可爱的蓝发少女" （太短、缺细节）
- ✅ 外貌: "一头及肩的冰蓝色短发，发尾微微内扣。琥珀色的杏眼，眼角微微上挑。瓜子脸，肤色白皙，身高约160cm，体型纤细。左耳戴着一枚星月形状的银色耳坠。"

---

## Validation Rules

| 规则ID | 检查项 | 条件 | 失败动作 |
|--------|--------|------|---------|
| V02-01 | 角色数 ≥ 2 | count ≥ 2 | RETRY |
| V02-02 | 每个角色的"外貌"字段 ≥ 100字符 | 逐角色检查 | RETRY |
| V02-03 | 每个角色含至少3个颜色关键词（正则: `[蓝红黑白金棕银绿紫粉青]色?`） | 逐角色检查 | WARN |
| V02-04 | 每个角色含"参考图提示词"且 ≥ 30字符 | 字段存在+长度 | RETRY |
| V02-05 | 角色名为纯中文（不含英文/数字/符号） | 正则: `^[\u4e00-\u9fff·]{2,5}$` | WARN |
