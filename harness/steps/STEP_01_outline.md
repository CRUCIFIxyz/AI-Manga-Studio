# STEP 01: 剧本大纲 (Outline)

> 流水线第1步。根据题材和用户创意生成世界观、故事梗概和分集大纲。

---

## Input Variables

| 变量 | 来源 | 类型 | 必填 |
|------|------|------|:--:|
| `{genre_cn}` | 用户选择 | 题材中文名 | ✅ |
| `{idea}` | 用户输入 | 创意文本 | ✅ |
| `{episode_count}` | 用户配置 | 整数 (1-10) | ✅ |

---

## System Prompt Template

```
你是一位专精于「{genre_cn}」题材的AI短剧编剧。

根据以下用户创意，生成剧本大纲。

【用户创意】
{idea}

【硬性要求】
- 共 {episode_count} 集，每集1-1.5分钟
- 世界观必须有具体的规则/体系描述（不能泛泛而谈）
- 故事梗概必须包含完整四段式：开端→冲突升级→高潮→结局
- 每集大纲结尾必须标注 [HOOK]，写一句具体的悬念（不是"接下来会发生什么？"这种泛泛写法）
- 严格按以下格式输出，不要输出任何寒暄语
```

---

## Output Format

```
### 世界观
（2-3句话。必须包含：世界背景 + 核心规则/力量体系/特殊设定。不能只是"这是一个魔法世界"的泛泛描述）

### 故事梗概
（4-6句话。必须覆盖：开端→冲突升级→高潮→结局。让读者看完能复述整个故事线）

### 分集大纲
**第1集**：（3-5句话核心情节——谁、在哪里、发生了什么、结果如何）
  → [HOOK]（1句具体悬念——描述一个具体的画面或台词，让观众迫切想看下一集）

**第2集**：（同上格式）
```

**反例**（不合格）：
- ❌ `[HOOK] 接下来会发生什么？` （太泛）
- ✅ `[HOOK] 她推开石门，黑暗中一双金色的眼睛缓缓睁开了。` （具体画面）

---

## Validation Rules

| 规则ID | 检查项 | 条件 | 失败动作 |
|--------|--------|------|---------|
| V01-01 | 生成集数与用户配置一致 | count == {episode_count} | RETRY |
| V01-02 | "世界观"段存在且 ≥ 50字符 | len ≥ 50 | RETRY |
| V01-03 | "故事梗概"段存在且 ≥ 100字符 | len ≥ 100 | RETRY |
| V01-04 | 每集结尾含 `[HOOK]` 标记 | 逐集检查，命中率=100% | RETRY |
| V01-05 | 每集大纲 ≥ 40字符（有意义内容） | 逐集检查 | RETRY |

---

## Pass Output

```json
{
  "outline_summary": "前200字摘要",
  "episode_outlines": ["第1集...", "第2集...", ...],
  "hook_positions": [1, 2, ...],
  "raw_content": "完整Markdown文本"
}
```
