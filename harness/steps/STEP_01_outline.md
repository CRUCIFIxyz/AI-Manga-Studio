# STEP 01: 剧本大纲 (Outline)

> 流水线第1步。根据题材和用户创意生成世界观、故事梗概和分集大纲。

---

## Input Variables

| 变量 | 来源 | 类型 | 必填 |
|------|------|------|:--:|
| `{genre_cn}` | 用户选择 | 题材中文名 | ✅ |
| `{idea}` | 用户输入 | 创意文本 | ✅ |
| `{episode_count}` | 用户配置 | 整数 (1-10) | ✅ |

---

## System Prompt Template

```
你是一位专精于「{genre_cn}」题材的AI短剧编剧。

根据以下用户创意，生成剧本大纲。

【用户创意】
{idea}

【要求】
- 共 {episode_count} 集，每集1-1.5分钟
- 以下格式输出，不要输出其他内容
```

---

## Output Format

```
### 世界观
（2-3句话：故事世界背景、核心规则、力量体系或特殊设定）

### 故事梗概
（4-6句话：完整故事线——开端→冲突→高潮→结局）

### 分集大纲
**第1集**：（3-5句话核心情节）→ [HOOK]（1句悬念）
**第2集**：（3-5句话核心情节）→ [HOOK]（1句悬念）
...
```

---

## Validation Rules

| 规则ID | 检查项 | 条件 | 失败动作 |
|--------|--------|------|---------|
| V01-01 | `episode_count` 与实际生成集数一致 | count == {episode_count} | RETRY |
| V01-02 | "世界观"段存在且 ≥ 50 字符 | len ≥ 50 | RETRY |
| V01-03 | "故事梗概"段存在且 ≥ 100 字符 | len ≥ 100 | RETRY |
| V01-04 | 每集结尾含 `[HOOK]` 或 `「` 标记 | 逐集检查，命中率=100% | RETRY |
| V01-05 | 每集大纲 ≥ 40 字符 | 逐集检查 | RETRY |

---

## Pass Output

校验通过后提取以下数据传给下游：

```json
{
  "outline_summary": "前200字摘要",
  "episode_outlines": ["第1集...", "第2集...", ...],
  "hook_positions": [1, 2, ...],
  "raw_content": "完整Markdown文本"
}
```
