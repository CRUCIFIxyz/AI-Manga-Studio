# STEP 05: 分镜脚本 (Storyboard)

> 流水线第5步。**一致性关键节点**——必须引用前面模块的角色名和场景名，不允许出现新角色。

---

## Input Variables

| 变量 | 来源 | 类型 | 必填 |
|------|------|------|:--:|
| `{genre_cn}` | 用户选择 | 题材中文名 | ✅ |
| `{episode_count}` | 用户配置 | 集数 | ✅ |
| `{character_appearances}` | STEP 02 输出 | 角色外貌Map | ✅ |
| `{scene_descriptions}` | STEP 03 输出 | 场景描述Map | ✅ |
| `{dialogue_summary}` | STEP 04 输出 | 台词摘要 | ✅ |

---

## System Prompt Template

```
你是一位分镜师，专精于「{genre_cn}」题材。

根据以下素材，写出逐镜分镜表。关键要求：分镜中的角色名和场景名必须与下方列表完全一致——禁止改名，禁止添加新角色。

【角色外貌（分镜中只能使用以下角色）】
{character_appearances}

【场景描述（分镜中只能使用以下场景）】
{scene_descriptions}

【台词摘要】
{dialogue_summary}

【硬性要求】
- 共 {episode_count} 集，每集5-8个镜头
- 表格格式：镜号 | 景别 | 画面描述 | 运镜 | 时长 | 转场
- 画面描述中必须使用上述角色名和场景名——引用时请用全名
- 景别只能从[全景/中景/近景/特写]中选择
- 运镜只能从[推/拉/摇/移/跟/静止]中选择
- 不允许在分镜中创造新角色名或场景名
```

---

## Output Format

```
### 第1集分镜

| 镜号 | 景别 | 画面描述 | 运镜 | 时长 | 转场 |
|------|------|---------|------|------|------|
| S101 | 全景 | 晨光中，{角色名}站在{场景名}前。金色阳光从左侧窗格斜射入内，在地面投下平行的光带。{角色名}的身影被拉得很长。 | 静止 | 4s | 淡入 |
| S102 | 近景 | ... | 缓慢推近 | 3s | 切 |
（继续S103~S108）

### 第2集分镜
...
```

---

## Validation Rules

| 规则ID | 检查项 | 条件 | 失败动作 |
|--------|--------|------|---------|
| V05-01 | 集数与 {episode_count} 一致 | count == {episode_count} | RETRY |
| V05-02 | 每集 ≥ 5 个镜头（表格行数） | 逐集检查 | RETRY |
| V05-03 | 镜头描述中至少引用1个角色名（来自STEP02） | 逐集检查 | RETRY |
| V05-04 | 景别在[全景/中景/近景/特写]范围内 | 逐镜检查 | WARN |
| V05-05 | 运镜在[推/拉/摇/移/跟/静止]范围内 | 逐镜检查 | WARN |
| V05-06 | 没有出现STEP02以外的角色名 | 新角色名=0 | RETRY |
| V05-07 | 使用了表格格式（含 `|` 分隔符） | 至少2行表格 | RETRY |
