# AI漫剧剧本生成器 MVP — 完整业务流程文档

> 版本：v3.2 | 日期：2026-07-23 | 项目：全球数字创业 第二阶段 MVP

---

## 一、系统概览

### 1.1 产品一句话

用户选择题材+画风+目标平台 → 输入创意关键词 → AI自动生成**6模块标准化剧本包** → 内容可直接复制到即梦/Pika等AI工具进行视频制作。

### 1.2 核心价值主张

> "AI短剧剧本界的Canva"——选格式、填想法、拿成品。不用再做"翻译"工序。

### 1.3 三大壁垒（对齐源文件pt.2 UVP）

| 壁垒 | 实现方式 |
|------|---------|
| **AI原生格式** | 6模块结构化输出，角色-分镜-提示词链式引用，即拿即用 |
| **工业化产出** | 一次API调用完成全部生成（~60秒），日产理论5-10套 |
| **版权保障** | System Prompt强约束：100%原创，不引用任何现有IP |

## 三、Step by Step 详细流程

### Step 1：选择题材（6选1）

用户从题材卡片中选择一个主题类型。

| 题材 | 英文标识 | 典型元素 |
|------|---------|---------|
| 🏫 校园奇幻 | campus_fantasy | 魔法学院、异能觉醒、契约兽 |
| 🏙️ 都市逆袭 | urban_revenge | 重生、商战、身份反转 |
| 🗡️ 古风仙侠 | xianxia | 修仙、门派、飞升渡劫 |
| 🤖 科幻末世 | sci_fi | 废土求生、机甲、AI觉醒 |
| 🔍 悬疑推理 | mystery | 密室、反转、侦探解谜 |
| 💕 甜宠现言 | romance | 总裁、契约婚姻、先婚后爱 |

**System Prompt 行为**：题材名注入 System Prompt 角色定位段——`specializing in "校园奇幻" genre`，AI据此调整世界观基调和剧情范式。

---

### Step 2：描述创意（自由文本）

用户在文本框中自由输入创意关键词。支持：
- 完整句子：`一只会说话的猫在魔法学院发现了地下王国`
- 关键词组合：`机甲少女 / 末世废土 / 寻找希望`
- 快捷填充：点击"🐱会说话的猫""🔄重生逆袭"等提示标签自动填入

**校验规则**：创意描述不可为空（前端+后端双重校验）。

---

### Step 3：配置参数

| 参数 | 选项 | 默认 | 说明 |
|------|------|------|------|
| **集数** | 1-10集 | 5集 | 每集1-1.5分钟时长 |
| **画风** | 10种 | 吉卜力 | 统一应用于所有视觉描述和AI提示词 |

**10种画风列表**：

| # | 画风 | 视觉关键词 |
|---|------|-----------|
| 1 | 🎨 吉卜力 | 水彩质感、温暖怀旧光、手绘感、治愈色调 |
| 2 | 🌅 新海诚 | 照片级光影、绚丽天空、镜头光晕、电影调色 |
| 3 | 🎌 日系二次元 | 赛璐璐着色、高饱和、大眼、动态线、2D动画感 |
| 4 | 📱 韩系漫画 | 修长比例、柔光渐变、光泽发质、数字绘画 |
| 5 | 🏰 美式卡通 | Disney/Pixar 3D、夸张表情、弹性质感 |
| 6 | 🎥 写实电影感 | 照片级纹理、戏剧光、电影颗粒、浅景深 |
| 7 | 🌃 赛博朋克 | 紫青霓虹灯、暗雨城、全息UI、体积雾 |
| 8 | 🏮 国风古韵 | 水墨笔触、缥缈云雾、丝绸质感、古典意境 |
| 9 | 📖 治愈绘本 | 柔和粉彩、圆润形状、蜡笔水彩质感 |
| 10 | 👾 像素复古 | 8-bit/16-bit、限定色盘、抖动效果 |

**System Prompt 行为**：每种画风含三个维度注入Prompt：
- `desc`（英文描述）→ 注入视觉风格要求、AI提示词格式模板
- `negative`（负面提示词）→ 注入每个平台的Negative prompt字段
- `name_cn`（中文名）→ 用于前端显示

---

### Step 4：选择目标平台（单选×2）

这是MVP核心差异化功能——**一个图像平台 + 一个视频平台**，各选一个。

#### 4.1 图像生成平台（4选1）

| 平台 | 专属Prompt格式 |
|------|---------------|
| **即梦/Flux** | `[画风], [角色], [动作], [场景], [光线], [画质] --ar 16:9` + Negative prompt |
| **Midjourney** | `[角色] in [场景], [动作], [画风] --ar 16:9 --style expressive --niji 6` + `--no` |
| **HappyHorse** | `Scene/Character/Action/Style/Mood/Camera` 结构化字段 + 参考图生成建议 |
| **Stable Diffusion** | `([画风]), [角色], [动作], [场景]` + 标准负面提示词模板（bad anatomy等） |

#### 4.2 视频生成平台（6选1）

| 平台 | 专属Prompt格式 | 核心技术参数 |
|------|---------------|-------------|
| **即梦（图生视频）** | 参考图 + 动作 + 运镜 | Motion strength (1-10) |
| **Pika/PixVerse** | 角色 + 运镜+动作 + 场景 | Motion intensity (low/med/high) |
| **Kling/可灵** | 首帧→尾帧 + 中间动作 | START/END FRAME + ACTION |
| **Hailuo/海螺** | 场景+角色+动作 | Camera类型枚举 |
| **HappyHorse（视频）** | Element1+2+3 + 互动定义 | Animation type分层控制 |
| **Runway Gen-3** | Base prompt + Motion Brush分区域 | Region 1/2/3逐区运动描述 |

**Prompt强力约束**（v3.2新增）：
- `**CRITICAL: ALL prompts MUST follow this exact format. No exceptions.**`
- `For EVERY shot, output EXACTLY:`
- `KEY RULES:`（每个平台3-5条专属规则）
- 提示词数量要求从3-5提升到5-8

---

### Step 5：AI生成（~60秒）

点击"生成剧本"按钮后：

#### 5.1 前端动画阶段

| 阶段 | 视觉反馈 | 时长 |
|------|---------|------|
| 页面切换 | 面板从输入页切换到生成页 | 0.4s fade |
| 进度条 | 流光shimmer动画，从0%→95%（分段加速） | 持续 |
| 步骤节点 | 6个节点依次点亮（灰色→激活→完成✓） | 同步进度 |
| 右侧清单 | ○→◉→✓ 状态同步更新 | 同步进度 |
| 状态文字 | 依次显示"正在调用DeepSeek→构建世界观→设计角色..." | 每阶段切换 |

#### 5.2 后端处理阶段

```
1. 参数校验
   ├── 创意描述非空
   ├── 题材类型在GENRE_MAP中
   ├── 集数1-10
   └── 平台值合法

2. 构建System Prompt（~5KB）
   ├── 角色定位+题材
   ├── 画风描述
   ├── 6模块输出模板（预计算join值）
   ├── 选中平台专属格式（_build_platform_formats单平台生成）
   ├── 硬性约束（7条MUST FOLLOW）
   └── 模块间交叉引用指令

3. 构建User Prompt
   ├── 用户创意原文
   ├── 题材/集数上下文
   └── 格式遵循指令

4. 调用DeepSeek Chat API
   ├── POST https://api.deepseek.com/v1/chat/completions
   ├── model: deepseek-chat
   ├── temperature: 0.8
   ├── max_tokens: 8192
   └── timeout: 180s

5. 解析返回Markdown
   ├── 正则拆分 `## 0X` 标题
   ├── 失败回退：按中文关键词手动拆分
   └── 兜底：整段作为完整剧本

6. 保存文件
   ├── 文件夹名：{题材}_{关键词前20字}_{时间戳}
   ├── 6个.md文件（UTF-8编码）
   └── 路径：输出剧本/{文件夹名}/

7. 返回JSON
   ├── success: true
   ├── folder/folder_path
   └── modules: {01_剧本大纲: "...", 02_角色设定: "...", ...}
```

#### 5.3 进度→结果过渡

- 进度条快速从95%跳到100%
- 所有步骤节点标记为done（绿色+✓）
- 0.6s延迟后切换到结果面板

### Step 6：使用结果

#### 6.1 6模块Tab查看

结果面板展示6个Tab标签，点击切换查看对应模块。Tab下方有滑动指示器。

| Tab | 内容 | 语言 |
|-----|------|------|
| ① 剧本大纲 | 世界观+故事梗概+分集大纲（每集含[HOOK]）| 中文 |
| ② 角色设定 | 姓名/年龄/外貌/服装/性格/标志动作/参考图提示词 | 中文 |
| ③ 场景描述 | 时间/地点/画面描述/氛围/参考图提示词 | 中文 |
| ④ 分集台词 | 场景标题+角色：台词（动作）+【本集钩子】 | 中文 |
| ⑤ 分镜脚本 | 镜号/景别/画面/运镜/时长/转场 表格 | 中文 |
| ⑥ AI提示词 | 选中平台的专属英文prompt（直接复制可用） | **英文** |

#### 6.2 快捷操作

| 按钮 | 功能 | 位置 |
|------|------|------|
| 📋 复制全部 | 将所有模块拼接后复制到剪贴板 | 顶部操作栏 |
| 📥 下载Markdown | 下载为单个.md文件 | 顶部操作栏 |
| 🔄 新建剧本 | 清空结果，返回输入页 | 顶部操作栏 |
| 🎨 复制AI提示词 | 仅复制模块06 | 右侧快捷面板 |
| 🎬 复制分镜脚本 | 仅复制模块05 | 右侧快捷面板 |
| 📖 复制剧本大纲 | 仅复制模块01 | 右侧快捷面板 |

#### 6.3 后续使用流程（生成后）

```
① 角色设定 → 复制"参考图提示词" → 粘贴到即梦/Midjourney → 生成角色立绘
② 场景描述 → 复制"参考图提示词" → 粘贴到即梦/Flux → 生成场景背景
③ AI提示词 → 复制对应平台prompt → 粘贴到目标工具 → 生成分镜图/视频
④ 分镜脚本 → 参照镜头表 → 在剪辑软件（剪映/PR）中按顺序拼接
⑤ 分集台词 → 复制到ElevenLabs/TTS → 生成配音
⑥ 后期合成 → 导入动画片段+配音+BGM → 添加字幕 → 导出MP4
```

## 四、数据流全景

```
┌─────────────┐     ┌──────────────┐     ┌───────────────┐     ┌──────────────┐
│  index.html │────▶│  Flask /gen  │────▶│ DeepSeek API  │────▶│  输出剧本/    │
│  (浏览器)    │     │  (后端路由)   │     │  (AI推理)     │     │  (磁盘文件)   │
└─────────────┘     └──────────────┘     └───────────────┘     └──────────────┘
       │                    │                     │                     │
       ▼                    ▼                     ▼                     ▼
  用户填表+选平台      参数校验+构建Prompt    System+User Prompt    6个.md文件
  genre/idea/ep        注入画风+平台格式      返回Markdown文本       UTF-8编码
  art_style/img/vid    temperature=0.8        max_tokens=8192       命名: {题材}_{关键词}_{时间戳}
```

### 4.1 System Prompt 构建流程

```
build_system_prompt()
  ├── 1. 查画风字典 art_styles[art_style]
  │     └── 获取 desc, negative, name_cn
  ├── 2. 预计算模板片段（避免f-string语法限制）
  │     ├── ep_outlines（分集大纲模板）
  │     ├── ep_scripts（分集台词模板）
  │     └── ep_storyboards（分镜表格模板）
  ├── 3. 调用 _build_platform_formats()
  │     ├── 查 img_formats[img_platform] → 获取图像平台专属格式
  │     └── 查 vid_formats[vid_platform] → 获取视频平台专属格式
  └── 4. 拼装完整 System Prompt → 返回字符串
```

### 4.2 API调用流程

```
call_deepseek_api()
  ├── url: {DEEPSEEK_API_BASE}/chat/completions
  ├── headers: Authorization: Bearer {API_KEY}
  └── payload:
       ├── model: deepseek-chat
       ├── messages: [{role:system, content}, {role:user, content}]
       ├── temperature: 0.8
       ├── max_tokens: 8192
       └── stream: false
```

### 4.3 解析与保存流程

```
API返回 content (Markdown文本)
  │
  ▼
parse_modules(content)
  ├── 正则: r"##\s*(0\d)\s*[^#\n]+"
  ├── 拆分为 dict: {"01": "内容1", "02": "内容2", ...}
  ├── 映射文件名: {"01": "01_剧本大纲", ...}
  └── 失败回退: 按中文关键词手动拆分
  │
  ▼
save_modules(modules, folder_name)
  ├── 创建文件夹: 输出剧本/{题材}_{关键词}_{时间戳}/
  ├── 逐个写入 .md 文件
  └── 返回文件夹路径
  │
  ▼
返回 JSON → 前端展示
```

## 五、平台适配矩阵

### 5.1 图像平台 × 画风组合

每个平台的prompt格式中注入的画风信息不同：

| 平台 | 画风注入方式 |
|------|-------------|
| 即梦/Flux | `[Art style: {art_desc}]` 作为prompt前缀 |
| Midjourney | `{art_desc}` 嵌入主体描述中 |
| HappyHorse | `Style: {art_desc}` 独立字段 |
| Stable Diffusion | `({art_desc})` 括号包裹作为主描述 |

### 5.2 视频平台核心技术参数

| 平台 | 核心参数 | 约束 |
|------|---------|------|
| 即梦视频 | Motion strength (1-10) | 推荐5-7，避免极端运动导致失真 |
| Pika | Motion intensity (low/med/high) | 必须显式声明运镜+角色动作方向 |
| Kling | START/END FRAME + ACTION | 首帧和尾帧必须描述不同姿态 |
| Hailuo | Camera类型枚举 | 必须从7种运镜中显式选择 |
| HappyHorse视频 | Animation type分层 | 必须列出所有Element并定义哪些动/哪些静 |
| Runway | Motion Brush 3+区域 | 每个区域必须有具体运动描述 |

## 六、安全与合规

| 措施 | 实现 |
|------|------|
| API Key保护 | `.env`文件存储，`.gitignore`排除，前端不可见 |
| 后端代理 | Flask路由转发，浏览器F12看不到API密钥 |
| 版权原创 | System Prompt硬性约束："100% original, no existing IP" |
| 输入校验 | 空输入拦截、题材白名单、集数范围限制 |
| 错误处理 | 超时504、API异常502、内部错误500，均有友好中文提示 |
| 编码规范 | 所有文件UTF-8编码，无乱码 |

## 七、语言切换

前端右上角🌐按钮支持中英文切换。切换影响：
- UI文字（按钮、标签、提示）
- placeholder文本
- Toast提示
- **不影响**生成的剧本内容（01-05始终中文，06始终英文）
