# AI漫剧剧本生成器 MVP 开发规范

> 版本：v1.0 | 日期：2026-07-23 | 项目：全球数字创业 第二阶段

---

## 一、项目概述

### 1.1 产品定位
AI漫剧标准化剧本"一键生成器"——用户输入题材+创意想法 → 输出一整套可直接用于即梦/Pika/PixVerse等AI视频工具的标准化剧本包。

### 1.2 核心价值
- **AI原生格式**：输出内容内置五大模块（大纲/人设/台词/分镜/AI提示词），打开就能跑
- **即梦/Pika就绪**：⑥AI提示词模块生成可直接复制的英文prompt
- **批量产出**：一次API调用产出完整剧本包

## 四、API规范

### 4.1 DeepSeek API配置
- **API Base URL**: `https://api.deepseek.com/v1`
- **Model**: `deepseek-chat`
- **API Key**: 从 `.env` 文件读取，绝不硬编码
- **请求方式**: POST `/v1/chat/completions`
- **超时时间**: 120秒（剧本生成长文本需要较长时间）
- **Temperature**: 0.8（保持创意性）
- **Max Tokens**: 8192

### 4.2 .env文件格式
```
DEEPSEEK_API_KEY=YOUR_API_KEY_PLACEHOLDER
DEEPSEEK_API_BASE=https://api.deepseek.com/v1
```

### 4.3 API调用流程
```
用户提交表单 → Flask接收 → 构建System Prompt → 
调用DeepSeek API → 流式/同步返回 → 解析JSON → 
保存Markdown文件 → 返回HTML展示
```

## 五、生成内容规范

### 5.1 System Prompt设计原则
1. 明确要求AI输出结构化Markdown格式
2. 每个模块用 `## 模块名` 分隔
3. ⑥AI提示词必须输出**英文prompt**（适配即梦/Pika等工具）
4. 每集留悬念/钩子（"预知后事如何"）
5. 单集时长适配1-1.5分钟（约300-500字台词）

### 5.2 六大模块输出格式

#### ① 剧本大纲 (01_剧本大纲.md)
```
- 世界观设定（100-200字）
- 故事梗概（200-300字）
- 分集大纲（每集50-80字 + 结尾钩子）
```

#### ② 角色设定 (02_角色设定.md)
```
- 每个角色：名称 / 年龄 / 外貌 / 服装 / 性格 / 标志动作
- 至少包含主角 + 1个配角
- 可附带Midjourney风格参考关键词
```

#### ③ 场景描述 (03_场景描述.md)
```
- 关键场景列表
- 每个场景：名称 / 时间 / 地点 / 画面描述 / 氛围
- 格式参考：即梦的"场景+氛围+风格"格式
```

#### ④ 分集台词 (04_分集台词.md)
```
- 按集划分
- 每集标注：场景名 / 角色名：台词 / （动作描写）
- 单集台词控制在300-500字
```

#### ⑤ 分镜脚本 (05_分镜脚本.md)
```
- 镜头号 / 景别（远景/全景/中景/近景/特写）
- 画面描述 / 运镜（推拉摇移跟升降）
- 时长（秒）/ 转场方式
- 单集5-8个分镜
```

#### ⑥ AI提示词 (06_AI提示词.md)
```
- 每个关键镜头一个英文prompt
- 格式：{画风} + {角色描述} + {动作} + {场景} + {光线} + {画质参数}
- 示例："Anime style, Studio Ghibli, a talking black cat wearing wizard hat, 
  sitting on wooden desk, sunlit classroom, cherry blossoms outside window, 
  soft warm lighting, clean lines, vibrant colors --ar 16:9"
```

### 5.3 命名规范
- 文件夹名：`{主题类型}_{核心要素}_{时间戳}`
- 示例：`校园奇幻_会说话的猫_20260723_143022`
- 文件名：`{序号}_{模块中文名}.md`
- 编码：UTF-8（无BOM）

## 六、前端UI规范

### 6.1 设计参考
参考用户提供的Dashboard设计图，采用：
- **卡片式布局**：每个模块用独立卡片展示
- **侧边栏导航**：左侧步骤导航（填入信息 → 生成中 → 查看结果）
- **进度可视化**：生成过程中展示步骤节点的动画进度条
- **分层信息展示**：关键指标卡片 + 详细内容区

### 6.2 配色方案（马卡龙色系 v3.0）
```css

```

### 6.3 动画效果要求
1. **页面加载**：卡片从下方淡入上滑（staggered animation）
2. **表单交互**：输入框focus时边框颜色渐变 + 微缩放
3. **提交按钮**：hover波纹效果 + 点击缩放反馈
4. **生成过程**：
   - 进度条带流光动画（shimmer effect）
   - 6个步骤节点依次点亮（sequential pulse）
   - 当前步骤显示旋转loading指示器
   - 背景粒子漂浮效果（Canvas实现）
5. **结果展示**：
   - Tab切换带滑动指示器
   - 内容区从右淡入
   - 复制按钮hover变色 + 复制成功提示toast
6. **下载按钮**：下载完成有飞入动画

## 七、代码规范（八荣八耻）

### 7.1 总则
1. **以暗猜接口为耻，以认真查阅为荣** → 调用API前查阅DeepSeek官方文档
2. **以模糊执行为耻，以寻求确认为荣** → 不确定时向用户确认后再动手
3. **以盲想业务为耻，以人类确认为荣** → 业务逻辑基于源文件pt.2中的真实痛点和数据
4. **以创造接口为耻，以复用现有为荣** → 使用Flask标准路由、DeepSeek标准API格式
5. **以跳过验证为耻，以主动测试为荣** → 每次代码改动后启动Flask验证，检查API连通性
6. **以破坏架构为耻，以遵循规范为荣** → 保持三层分离：路由/API调用/文件保存
7. **以假装理解为耻，以诚实无知为荣** → API报错时报真实错误信息
8. **以盲目修改为耻，以谨慎重构为荣** → 改动前先理解现有代码逻辑

### 7.2 Python代码风格
- 缩进：4空格
- 命名：函数用蛇形，类用驼峰，常量全大写
- 类型注解：关键函数使用类型提示
- 错误处理：try-except包裹API调用，返回友好错误信息
- 注释：遵循用户偏好——零emoji、不注释空行、语言专业、按逻辑块组织

### 7.3 前端代码风格
- CSS使用CSS Variables管理配色
- JS使用async/await处理异步请求
- 动画使用CSS animation + requestAnimationFrame
- 避免内联样式（除Jinja2动态渲染外）
